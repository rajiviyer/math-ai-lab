# Personal Computational Mathematics Laboratory

## Vision

This project is a Personal Computational Mathematics Laboratory.

Its purpose is to help me develop deep mathematical intuition by building concepts from first principles through code, simulations, visualizations, and experimentation.

This repository is intended to become a living Mathematical Operating System: a collection of computational instruments that I understand deeply because I built them myself and can use whenever a real-world problem appears.

The objective is not merely to solve problems, but to build reusable mental models and executable models.

---

## Core Philosophy

Mathematics should be:

Concept → Simulation → Visualization → Experimentation → Intuition → Reusable Tool → Real-World Application

Every module should help answer:

* Why does this happen?
* What assumptions are being made?
* What changes when parameters change?
* Where does the model fail?
* How can it be generalized?
* What real-world problems does this apply to?

---

## Learning Principles

Prefer:

* First-principles thinking
* Simulations over memorization
* Interactive exploration
* Visual intuition
* Small experiments
* Incremental complexity
* Discovering patterns, symmetries, and invariants
* Building reusable tools and abstractions

Avoid:

* Treating mathematics as formulas to memorize
* Black-box implementations
* Premature optimization
* Excessive abstraction that obscures understanding

---

## Role of AI

AI is a collaborator, tutor, and accelerator.

Use AI to:

* Generate boilerplate code
* Create visualizations and widgets
* Suggest experiments
* Explain concepts
* Derive mathematics
* Review implementations
* Propose extensions and real-world applications

Do not let AI replace reasoning.

Whenever possible:

1. Make predictions first.
2. Implement core logic yourself.
3. Run experiments.
4. Visualize the results.
5. Use AI to deepen understanding and generalize insights.

---

## Design Principles

Every concept should ideally contain:

* Explanation
* Mathematical derivation
* Assumptions
* Simulation code
* Visualizations
* Interactive controls
* Experiments
* Real-world examples
* Edge cases
* Limitations
* Related concepts
* Extensions and open questions

---

## Preferred Architecture

Build small, composable instruments.

Add to or Improve the existing project structure.

Each module should be usable independently and composable with others.

---

## Engineering Guidelines

Prefer:

* Clear and readable code
* Small focused functions
* Explicit assumptions
* Reproducible simulations
* Well-documented experiments
* Parameterized notebooks and utilities
* Interactive visualizations where useful

Avoid:

* Monolithic notebooks
* Hidden state
* Magic constants
* Clever but opaque implementations
* Implementations that obscure the mathematics

---

## Success Criteria

A module is successful if:

* I understand the concept more deeply after building it.
* I can explain it intuitively.
* I can experiment with it.
* I can reuse it to solve new problems.
* I can connect it to other mathematical ideas.
* I can apply it to real-world engineering, AI, data, or decision-making problems.

This repository should gradually evolve into a personal library of mathematical instruments and computational experiments that compound in value and deepen intuition over time.
